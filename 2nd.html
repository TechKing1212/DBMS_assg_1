DBMS Data Models & Architecture Explorer

Explore different data models with visual diagrams and see a live comparison of DBMS architectures.

Relational Model

Table-based
e.g., Student(RollNo, Name, Class)
  • Data in rows & columns
  • Keys define relationships
  • Uses SQL for queries

Hierarchical Model

Tree-structured
e.g., Org Chart: Manager → Team
  • Parent-child hierarchy
  • Fast for 1-to-many
  • Poor for many-to-many

Network Model

Graph-based
e.g., Student—>Course, Student—>Club
  • Allows complex links
  • Multiple parents/children
  • Flexible but complex

Document Model

Document/JSON Storage
e.g., NoSQL: MongoDB docs
  • Stores rich, nested docs
  • Schema-less flexibility
  • Great for web apps
DBMS Architecture Comparison
Centralized Architecture
  • Single server, all data/services
  • Clients access via network
  • Easy management, risk of bottleneck
Client → Central Server (DBMS) → Client
Client-Server Architecture
  • Separate clients & database servers
  • Scalable, less bottleneck
  • Supports multiple remote clients
Client 1 ⇄ Network ⇄ DB Server ⇄ Client 2
